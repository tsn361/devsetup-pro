# Security Policy

## Supported Versions

Security fixes are provided for the latest release on the `main` branch.

## Reporting a Vulnerability

If you believe youâ€™ve found a security vulnerability in DevSetup Pro, please report it responsibly.

- Preferred: email `security@devsetup.pro` (or if not available yet, open a GitHub issue with **minimal details** and request a private channel).
- Include:
  - Steps to reproduce
  - Impact assessment (what an attacker can do)
  - Affected OS / Ubuntu version / WSL details
  - Logs or screenshots with secrets removed

## What Not To Share Publicly

Please do not include any of the following in public issues:
- sudo passwords
- API keys/tokens
- private SSH keys
- full system logs containing usernames or sensitive paths

## Response Expectations

We aim to acknowledge reports within 72 hours and provide a fix or mitigation as soon as reasonably possible.
